1 Can Euler angles represent any rotation? T Yes — you can reach any orientation in 3D via 3 angles, but you may lose a DOF (gimbal lock) at certain configurations.
2 Is j + k a rotation quaternion? F A rotation quaternion must have unit norm. |j+k| = √2 ≠ 1.

6 If q, r are rotation quaternions then qr is also a rotation quaternion? T Product of two unit quaternions is unit. This is the composition of rotations.
7 Every color in sRGB space can be represented by 3 values? T sRGB is a 3-component color space (R, G, B).
8 Perspective projection can be described using only translation & scale? F Perspective is not an affine transformation — the homogeneous divide (division by w) makes it projective, not expressible as T·S alone.
9 Euler angles suffer from gimbal lock? T Classic problem — two rotation axes align at certain configurations, killing one DOF.
10 Some colors can't be represented using sRGB? T sRGB is a subset of visible colors. Very saturated colors (like some greens/blues) fall outside its gamut.
11 If q1, q2 are rotation quaternions, is (q1+q2)/2 a rotation quaternion? F ||(q1+q2)/2||² = (2 + 2cos θ)/4 = cos²(θ/2). This equals 1 only when θ=0. In general it's not unit-norm, so not a rotation quaternion.
Proof: norm² of q1+q2 = 2 + 2(w1w2+x1x2+y1y2+z1z2) = 2+2cos θ. Divided by 4 → cos²(θ/2) ≠ 1 generally.

13 Sorting transparent objects closest→furthest is less computationally expensive? T Closest-first is correct for back-to-front (painter's algorithm) blending but it is indeed less expensive than furthest-first since depth test early-outs eliminate more fragments.

14 Alpha blending is a type of view-based transparency? — Not explicitly in the provided key. Alpha blending IS order-dependent and view-based (depends on camera), so T is reasonable.
15 You can make a projection that enlarges far objects and shrinks near ones? — Yes — just flip the sign of the perspective depth. Mathematically valid. Likely T.

7 marks
Part 1 — Draw triangle (no steps needed)

Vertices: P₀=(2,0), P₁=(0,4), P₂=(5,4). Plot and rasterize using scanline/barycentric.

Part 2 — Barycentric coordinates of point (2,2) — steps required

Edge Functions

// f_i(p) = (p - v_i) × (v_next - v_i)  [2D cross product]
f0 = 20 - 5y        // opposite vertex P0=(2,0)
f1 = 3y - 4x + 8   // opposite vertex P1=(0,4)
f2 = 2y + 4x - 8   // opposite vertex P2=(5,4)

// evaluate at query point (2,2)
f0(2,2) = 20 - 5·2 = 10
f1(2,2) = 3·2 - 4·2 + 8 = 6 - 8 + 8 = 6
f2(2,2) = 2·2 + 4·2 - 8 = 4 + 8 - 8 = 4

// normalization: evaluate f0 at its opposite vertex P0=(2,0)
f0(2,0) = 20 - 5·0 = 20

Result

λ₀ = f0(2,2)/f0(2,0) = 10/20 = 0.5
λ₁ = f1(2,2)/f0(2,0) = 6/20  = 0.3
λ₂ = f2(2,2)/f0(2,0) = 4/20  = 0.2

Check: 0.5 + 0.3 + 0.2 = 1.0 ✓
Part 3 — Color at (2,2) given vertex colors
C₀ = (1,1,0)   λ₀ = 0.5
C₁ = (0,1,1)   λ₁ = 0.3
C₂ = (1,0,1)   λ₂ = 0.2

Color = 0.5·(1,1,0) + 0.3·(0,1,1) + 0.2·(1,0,1)
      = (0.5,0.5,0) + (0,0.3,0.3) + (0.2,0,0.2)
      = (0.7, 0.8, 0.5)
Part 4 — Perspective-corrected barycentric (w=4 for all vertices)

Perspective Correction Formula

// corrected λ_i = (λ_i / w_i) / Σ(λ_j / w_j)

All w's are equal (w=4), so:
λ_i/w_i = λ_i/4 for all i

Σ = (0.5 + 0.3 + 0.2)/4 = 1.0/4 = 0.25

corrected λ₀ = (0.5/4) / 0.25 = 0.5
corrected λ₁ = (0.3/4) / 0.25 = 0.3
corrected λ₂ = (0.2/4) / 0.25 = 0.2
When all w values are equal, perspective correction changes nothing — the w's cancel out.

Part 2 — Given model matrix M, find transformed attributes

Position

P_world = M · V₀

Multiply full 4×4 matrix by the homogeneous position.

Normal

N_world = (M⁻¹)ᵀ · N₀

Normals use the inverse-transpose to stay perpendicular after non-uniform scale.

Color

unchanged

Color is not a geometric attribute — model matrix doesn't affect it.

Texture Coords

unchanged (usually)

UV coords are defined in model space and passed through as-is unless a UV transform is applied separately.

Part 3 — Given projection matrix P, find HCS and NDC

Steps

HCS = P · V              // direct matrix multiply
NDC = (x/w, y/w, z/w)   // perspective divide
Part 4 — Is projection matrix P orthogonal?
A matrix is orthogonal if Mᵀ = M⁻¹ (columns are orthonormal unit vectors). The answer was yes because the given P had only scaling and no translation — a scaling-only matrix with ±1 values is orthogonal. Check: columns must have unit norm and be mutually perpendicular.
Q6

Barycentric with & without Perspective Correction

Without perspective correction
λ_i = f_i(p) / f_i(v_opposite)    // plain 2D barycentric
With perspective correction
// given homogeneous w values for each vertex
corrected_λ_i = (λ_i / w_i) / Σ_j(λ_j / w_j)
This corrects for the fact that linear interpolation in screen space ≠ linear interpolation in world space after perspective projection.
Normal at point (0,0)

Interpolate normals using corrected barycentric weights, then renormalize the result:

N = normalize(λ₀·N₀ + λ₁·N₁ + λ₂·N₂)

Hidden Surface Removal

Color at a point on the triangle

Backface culling — quad without being culled

Key Idea

A triangle is front-facing if its vertices are ordered counter-clockwise (CCW) in screen space (or NDC). For a quad (4 vertices), split into 2 triangles and verify each is CCW. If the winding order of your 4-vertex sequence produces CCW triangles, they won't be culled.

// CCW test: for triangle (A, B, C)
signed_area = (B-A) × (C-A)   // 2D cross product
if signed_area > 0 → CCW → front-facing → NOT culled
Viewport transform
// NDC → viewport (screen space)
x_vp = (x_ndc + 1) / 2 · width
y_vp = (y_ndc + 1) / 2 · height   // or flip Y depending on convention

Transformations + View Matrix

Part 1 — 90° rotation around Y, then Scale(1,2,1)

Rotation around Y by 90°

Ry(90°):
 cos90  0  sin90   →    0  0  1
    0   1     0    →    0  1  0
-sin90  0  cos90   →   -1  0  0

Then Scale(1,2,1) scales the Y column by 2.

M = S · R:
 0  0  1  0
 0  2  0  0
-1  0  0  0
 0  0  0  1

For normals, compute (M⁻¹)ᵀ and apply to normal vectors, then normalize.

Color and UV coords are unchanged by the model matrix.

Part 2 — View Matrix from eye, lookAt, up

Look-At Construction

f = normalize(lookAt - eye)        // forward
r = normalize(f × up)              // right
u = r × f                          // true up (reorthogonalized)

View =
| r.x   r.y   r.z  -dot(r, eye) |
| u.x   u.y   u.z  -dot(u, eye) |
|-f.x  -f.y  -f.z   dot(f, eye) |
|  0     0     0        1        |
The last column is the translation that moves world origin to camera. Notice -f (we look down -Z in OpenGL convention).
Part 3 — NDC and HCS given P
HCS = P · V_camera
NDC = HCS.xyz / HCS.w
Part 4 — Is projection orthographic?
If P has no off-diagonal terms that mix x/y/z with w (i.e. the last row is [0,0,0,1]), it's orthographic — no perspective divide happens. If P sets w to something other than the original w, it's perspective.
Part 5 — NDC → Viewport
x_vp = (x_ndc + 1) / 2 · W
y_vp = (y_ndc + 1) / 2 · H

Sutherland-Hodgman Clipping

Given Vertices & Colors

VertexPositionColor
V0(-2, 0, 0)(1, 0, 0)
V1(0, 0, 0)(0, 1, 0)
V2(0, 2, 2)(0, 0, 1)

Which planes clip?

NDC clip planes: x∈[-1,1], y∈[-1,1], z∈[-1,1]

V0 = (-2,0,0): x=-2 < -1  → OUTSIDE left
V1 = (0,0,0):  all in [-1,1] → INSIDE
V2 = (0,2,2):  y=2 > 1, z=2 > 1 → OUTSIDE top, back

Apply Sutherland-Hodgman — Left Plane (x = -1) first

Edge V0→V1: V0 outside, V1 inside
  → compute intersection with x=-1:
  t = (-1 - (-2)) / (0 - (-2)) = 1/2 = 0.5
  P_int = V0 + 0.5·(V1-V0) = (-2+1, 0, 0) = (-1, 0, 0)
  C_int = (1,0,0) + 0.5·((0,1,0)-(1,0,0)) = (0.5, 0.5, 0)
  Output: P_int, V1

Edge V1→V2: both checked — V1 inside, V2 inside (for left plane x=-1: V2.x=0 ≥ -1)
  Output: V2

After left clip:  [(-1,0,0,C=(0.5,0.5,0)), (0,0,0,C=(0,1,0)), (0,2,2,C=(0,0,1))]

Right Plane (x = 1)

All x values: -1, 0, 0 — all ≤ 1 → no clipping needed

Top Plane (y = 1)

(-1,0,0): y=0 ≤ 1 → inside
(0,0,0):  y=0 ≤ 1 → inside
(0,2,2):  y=2 > 1 → outside

Edge (0,0,0)→(0,2,2): inside→outside
  t = (1-0)/(2-0) = 0.5
  P_int = (0,1,1)
  C_int = (0,1,0) + 0.5·((0,0,1)-(0,1,0)) = (0, 0.5, 0.5)

Edge (0,2,2)→(-1,0,0): outside→inside
  t = (1-2)/(0-2) = 0.5
  P_int = (0+0.5·(-1-0), 2+0.5·(0-2), 2+0.5·(0-2)) = (-0.5, 1, 1)
  C_int = (0,0,1) + 0.5·((0.5,0.5,0)-(0,0,1)) = (0.25, 0.25, 0.5)

After top clip: [(-1,0,0), (0,0,0), (0,1,1), (-0.5,1,1)]

Back Plane (z = 1)

(-1,0,0): z=0 ≤ 1 → inside
(0,0,0):  z=0 ≤ 1 → inside
(0,1,1):  z=1 ≤ 1 → inside (on boundary = inside)
(-0.5,1,1): z=1  → inside

→ no further clipping
Final clipped polygon (4 vertices) after left + top clip. Bottom, right, front planes don't clip this particular triangle.
Q11

Texture Mapping — 4×4 Texture

Part 1 — Number of Mip levels
levels = floor(log2(max(width, height))) + 1
       = floor(log2(4)) + 1
       = 2 + 1 = 3   // levels: 4×4, 2×2, 1×1
Part 2 — Choose mip level for triangle

Method

// compute UV footprint: how many texels does one pixel cover?
d = max(||dUV/dx||, ||dUV/dy||)    // screen-space UV derivatives
level = log2(d · texture_size)

// pick level that gives ~1 texel per pixel
// for a coarse triangle covering a large screen area → higher level (smaller mip)

The exact answer depends on the given triangle's screen-space size vs texture size.

Part 3 — Bilinear at Level 0, Repeat mode

Steps

1. Take UV coords (u, v)
2. Apply REPEAT: u = frac(u), v = frac(v)   // wrap using modulo
3. Map to texel space: u' = u·(W-1), v' = v·(H-1)
4. Get 4 surrounding texels (floor/ceil on both axes)
5. Bilinear interpolate:
   top    = lerp(T[x0,y1], T[x1,y1], frac(u'))
   bottom = lerp(T[x0,y0], T[x1,y0], frac(u'))
   result = lerp(bottom, top, frac(v'))
Part 4 — Trilinear at Level 0, Clamp mode
The exam answer was none (no valid color). The reason: level 0 has 4×4 texels, level 1 has 2×2. Trilinear blends between two adjacent mip levels. At exactly level 0, there's no level -1 to blend with — so trilinear at the very bottom level is undefined/degenerate. The exam expected you to recognize this edge case.

Light & Ray Tracing — Blinn-Phong / Microfacet

Part 1 — Half vector h
// l = direction from surface point to light
// v = direction from surface point to eye
h = normalize(l + v)

The microfacet normal that caused the reflection is exactly h — the half-vector between incoming and outgoing directions.

Part 2 — Perfectly smooth surface — any light reaching eye?
Perfectly smooth → Dirac delta NDF (only one normal direction).
The reflected ray from a point light is a single direction.
// light reaches the eye only if the reflected direction exactly equals v
→ Yes, but only if reflect(-l, n) == v exactly.
→ In practice: extremely unlikely unless specifically constructed.
Answer: Yes — but only in one exact direction. A smooth surface acts like a perfect mirror — only the one specular ray reaches the eye.
Part 3 — Which α makes NDF uniform over northern hemisphere?
GGX NDF: D(m) ∝ α² / (cos²θ · (α²-1) + 1)²

For D to be constant (uniform), we need the θ-dependence to vanish.
When α=1: denominator = (cos²θ·0 + 1)² = 1 → D = constant ✓

Answer: α = 1
Part 4 — Blinn-Phong L₀
h = normalize(l + v)
L0 = Id · kd · max(0, n·l)  +  Is · ks · max(0, n·h)^shininess

// Id = diffuse light intensity, Is = specular
// kd, ks = material diffuse/specular coefficients
Part 5 — Shadow ray intersections (Triangle, Sphere, Sphere)
Shadow ray: origin = surface point P, direction = normalize(P_light - P)

Test each shape:
- Check if intersection t > 0 (in front of P) and t < dist(P, light)
- If yes → that shape blocks the light

List ALL shapes where this is true.

Without the specific geometry values from the exam, you test each shape analytically. The answer depends on the given vertex/sphere data.

Part 6 — Is point P in shadow?
P is in shadow if any shape intersects the shadow ray (t ∈ (0, dist_to_light)).
If Part 5 found at least one blocker → yes, P is shadowed.

Full Barycentric Pipeline — Triangle (0.5,0.5), (4.5,0.5), (2.5,4.5)

Part 1 — Rasterize using offscreen point (-1,-1)

Edge function setup

P0=(0.5,0.5), P1=(4.5,0.5), P2=(2.5,4.5)
offscreen = (-1, -1)  // to determine inside sign

For each pixel (x+0.5, y+0.5) in bounding box:
  compute f0, f1, f2
  if same sign as f_i(offscreen) for all i → pixel is inside → draw it
The offscreen point is used to determine which sign means "inside". If f_i(offscreen) > 0 for all i, then pixels where all f_i > 0 are inside.
Part 2 — Barycentric of (2.5, 2.5) without perspective correction
// Using the triangle area method
Total area  = 0.5 · |( P1-P0) × (P2-P0)|
            = 0.5 · |(4,0) × (2,4)| = 0.5 · 16 = 8

Sub-area for λ0 (opposite P0): triangle(P, P1, P2)
  = 0.5·|(P1-P)×(P2-P)| = 0.5·|(2,−2)×(0,2)| = 0.5·4 = 2
  λ0 = 2/8 = 0.25

Sub-area for λ1 (opposite P1): triangle(P0, P, P2)
  = 0.5·|(P−P0)×(P2−P0)| = 0.5·|(2,2)×(2,4)| = 0.5·|(8-4)| = 2
  λ1 = 2/8 = 0.25  (verify: sum should be 1)

λ2 = 1 - 0.25 - 0.25 = 0.5

Result: λ = (0.25, 0.25, 0.5)
Part 3 — With perspective correction (depth ~0.5 given)
// need w values for each vertex from HCS
corrected_λ_i = (λ_i / w_i) / Σ(λ_j / w_j)

// interpolated depth: z = λ0·z0 + λ1·z1 + λ2·z2 (use corrected λ)
Part 4 — Depth test: does (2.5,4.5) with some depth overdraw our point?
depth buffer keeps the fragment with smallest z (closest to camera).
if new_depth < stored_depth → overwrite (draw new fragment)
if new_depth >= stored_depth → discard
The point with smaller depth wins. The exam said depth ~0.5 — compare against the stored depth at that pixel, smallest one gets drawn.
Part 5 — Alpha blending (α=0.25, γ=2.2)
// alpha blend in linear space, not gamma-corrected space

1. Convert to linear:  C_linear = C_srgb ^ γ   (γ = 2.2)
2. Blend:
   C_out = α · Cs + (1−α) · Cd
         = 0.25 · Cs_linear + 0.75 · Cd_linear
3. Convert back to sRGB: C_final = C_out ^ (1/γ)
The exam expected you to do the gamma correction step — blending in sRGB space gives wrong results. Always blend in linear space.
